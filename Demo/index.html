TUI Framework

TUI 是一个强大的 Web UI 框架，提供了一组方便使用的控件，还实现了无脚本前后台交互，这里只是一个简单的介绍，没有把所有功能都讲到，由于本库目前还只限于内部使用，故暂时没有API文档，有问题请联系：秦诺

多语言支持

实现了多语言框架，并随库携带 en-us 和 zh-cn 两套翻译。用户可以提供自己的翻译脚本，只要注册一下就可以了，使用 tui.str("originText") 就可以得到当前语言环境下对应的翻译结果，如果没有找到则返回原始内容，也可以手工设定语言区域，这样可以自己控制要显示的语言。

数据表单

  • 表单可以用在很多地方，数据的增删改查都可以使用表单系统，TUI集成的表单和HTML原有的FORM并不相同， TUI-FORM使用分散的组织结构，并不要求组成数据都写入表单标签之内。这样设计页面的时候可以单独把所有表单标签放在一起， 组成一个交互区，也就是说所有向服务器发起请求的地方都在一起，更加便于维护。同时这么做也增强了灵活性，原有的FORM只能提交内部的控件数据，如果多个表单共用一些控件的时候会比较不方便。
  • TUI表单还实现了数据到JSON格式的自动序列化和反序列化功能，一个表单既可以完成数据的采集又可以作为返回数据的显示。

在表单 queryForm 中的data-target指定了返回的结果应该被显示到的目标对象， 这个对象可以是任何拥有 value 接口的 TUI 控件，或者是个普通的HTML标签， 如果是普通的HTML标签，则只是简单的把数据序列化为JSON字符串， 然后填充到 data-value 属性中。表单的data-target如果指定为另一个表单，则会把返回的数据分解到目标表单的各个控件中。
这就是表单控件的一个特性：无脚本交互，表单甚至可以嵌套使用（不要互相嵌套，这是死循环），这样可以实现内容的分组和组合。

表单代理

使用上面演示的表单已经很方便了，但是在有些情况下，或者是还不够灵活，或者是无法实现某种需求，这时就要用到强大的表单代理控件了，这里列举了如下几种情况需要使用到表单代理：
  • 当表单字段是一组单选或复选框时，此时由于表单字段对应的是单选组或复选组，所以简单的设置checkbox或radiobox的data-ajax-form和data-ajax-field属性是和需求不符的，这只会把改对象的值加入表单，而没有起到判定选择的作用。
  • 当希望取值的属性不是默认的"value"属性时，这时候使用表单代理可以指定希望使用的属性。
  • 当多个表单会复用某些控件的时候，这时由于一个控件只能设置一个ajax-form属性，所以只能隶属于一个表单，这时使用表单代理就可以映射该控件到不同的表单中。
  • 当表单字段的值需要取自一些内置控件，或需要更为精细的脚本控制时，这时使用绑定事件的表单代理控件可以以最少的代码实现最大化的灵活性。
  • 表单代理控件可以摆放到表单控件内部，这样代码看起来更加清晰，表单有哪些字段可以一目了然，并且放在里面的代理控件可以不用填写 data-ajax-form 属性，减少了一些代码。
